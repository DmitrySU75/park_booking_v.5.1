# Настройка двусторонней интеграции с 1С:УНФ

## 1. Настройка HTTP-сервиса в 1С:УНФ

### 1.1. Создание HTTP-сервиса

В конфигураторе 1С:УНФ:

1. Откройте ветку **«Общие»** → **«HTTP-сервисы»**
2. Нажмите **«Добавить»**
3. Заполните:
   - **Имя:** `booking`
   - **Шаблон:** `/booking/`
   - **Описание:** «Приём бронирований с сайта и 1С»

### 1.2. Создание шаблона URI

Внутри HTTP-сервиса `booking` добавьте шаблоны URI:

#### Шаблон для создания бронирования

| Параметр       | Значение          |
| -------------- | ----------------- |
| **Шаблон**     | `/create`         |
| **HTTP-метод** | `POST`            |
| **URL-шаблон** | `/booking/create` |

#### Шаблон для отмены бронирования

| Параметр       | Значение          |
| -------------- | ----------------- |
| **Шаблон**     | `/cancel`         |
| **HTTP-метод** | `POST`            |
| **URL-шаблон** | `/booking/cancel` |

#### Шаблон для изменения бронирования

| Параметр       | Значение          |
| -------------- | ----------------- |
| **Шаблон**     | `/change`         |
| **HTTP-метод** | `POST`            |
| **URL-шаблон** | `/booking/change` |

#### Шаблон для получения заказов

| Параметр       | Значение          |
| -------------- | ----------------- |
| **Шаблон**     | `/orders`         |
| **HTTP-метод** | `GET`             |
| **URL-шаблон** | `/booking/orders` |

### 1.3. Код обработчика создания бронирования

```javascript
Процедура ОбработкаЗапросаНаСозданиеБрони(Запрос, Ответ)

    ВходящиеДанные = ПрочитатьJSON(Запрос.ПолучитьТелоКакСтроку());

    // Проверка API-ключа
    Если ВходящиеДанные.api_key <> "секретный_ключ" Тогда
        Ответ.УстановитьКодСостояния(401);
        Ответ.УстановитьТелоИзСтроки(ЗаписатьJSON(Новый Структура("success", Ложь, "error", "Unauthorized")));
        Возврат;
    КонецЕсли;

    // Проверка обязательных полей
    Если ВходящиеДанные.resource_id = Неопределено
        Или ВходящиеДанные.start_time = Неопределено
        Или ВходящиеДанные.end_time = Неопределено
        Или ВходящиеДанные.client_name = Неопределено
        Или ВходящиеДанные.client_phone = Неопределено Тогда
        Ответ.УстановитьКодСостояния(400);
        Ответ.УстановитьТелоИзСтроки(ЗаписатьJSON(Новый Структура("success", Ложь, "error", "Missing required fields")));
        Возврат;
    КонецЕсли;

    // Создание контрагента
    Контрагент = НайтиИлиСоздатьКонтрагента(ВходящиеДанные.client_name,
        ВходящиеДанные.client_phone, ВходящиеДанные.client_email);

    // Создание номенклатуры (услуги)
    Номенклатура = НайтиИлиСоздатьНоменклатуру("Аренда беседки", ВходящиеДанные.resource_name);

    // Создание документа «Реализация услуг»
    Док = Документы.РеализацияУслуг.СоздатьДокумент();
    Док.Контрагент = Контрагент;
    Док.Дата = ТекущаяДата();
    Док.Комментарий = ВходящиеДанные.comment;

    Строка = Док.Услуги.Добавить();
    Строка.Номенклатура = Номенклатура;
    Строка.Количество = 1;
    Строка.Цена = ВходящиеДанные.total_price;
    Строка.Сумма = ВходящиеДанные.total_price;

    // Реквизиты для связи с сайтом
    Док.УстановитьРеквизит("ЗаказСайта", ВходящиеДанные.order_number);
    Док.УстановитьРеквизит("IDБронирования", ВходящиеДанные.librebooking_id);
    Док.УстановитьРеквизит("Беседка", ВходящиеДанные.resource_name);
    Док.УстановитьРеквизит("ВремяНачала", ВходящиеДанные.start_time);
    Док.УстановитьРеквизит("ВремяОкончания", ВходящиеДанные.end_time);

    Док.Записать();
    Док.Провести();

    // Формирование ответа
    Ответ.УстановитьКодСостояния(200);
    Ответ.УстановитьТелоИзСтроки(ЗаписатьJSON(Новый Структура(
        "success", Истина,
        "document_number", Док.Номер,
        "document_date", Док.Дата
    )));

КонецПроцедуры
```

### 1.5. Код обработчика изменения бронирования

```javascript
Процедура ОбработкаЗапросаНаИзменениеБрони(Запрос, Ответ)

    ВходящиеДанные = ПрочитатьJSON(Запрос.ПолучитьТелоКакСтроку());

    // Проверка API-ключа
    Если ВходящиеДанные.api_key <> "секретный_ключ" Тогда
        Ответ.УстановитьКодСостояния(401);
        Возврат;
    КонецЕсли;

    // Поиск документа
    Док = НайтиДокументПоЗаказуСайта(ВходящиеДанные.order_number);

    Если Док = Неопределено Тогда
        Ответ.УстановитьКодСостояния(404);
        Возврат;
    КонецЕсли;

    // Обновление времени
    Если ВходящиеДанные.new_start_time <> Неопределено Тогда
        Док.УстановитьРеквизит("ВремяНачала", ВходящиеДанные.new_start_time);
    КонецЕсли;

    Если ВходящиеДанные.new_end_time <> Неопределено Тогда
        Док.УстановитьРеквизит("ВремяОкончания", ВходящиеДанные.new_end_time);
    КонецЕсли;

    // Обновление беседки
    Если ВходящиеДанные.new_resource_id <> Неопределено Тогда
        Док.УстановитьРеквизит("Беседка", ВходящиеДанные.new_resource_name);
    КонецЕсли;

    // Пересчет цены, если изменилось время
    Если ВходящиеДанные.new_price <> Неопределено Тогда
        Если Док.Услуги.Количество() > 0 Тогда
            Док.Услуги[0].Цена = ВходящиеДанные.new_price;
            Док.Услуги[0].Сумма = ВходящиеДанные.new_price;
        КонецЕсли;
    КонецЕсли;

    Док.Записать();

    Ответ.УстановитьКодСостояния(200);
    Ответ.УстановитьТелоИзСтроки(ЗаписатьJSON(Новый Структура("success", Истина)));

КонецПроцедуры
```

### 1.6. Код обработчика получения заказов

```javascript
Процедура ОбработкаЗапросаНаПолучениеЗаказов(Запрос, Ответ)

    ВходящиеПараметры = Запрос.ПараметрыURL;

    // Проверка API-ключа
    Если ВходящиеПараметры["api_key"] <> "секретный_ключ" Тогда
        Ответ.УстановитьКодСостояния(401);
        Возврат;
    КонецЕсли;

    // Получение параметров
    НачалоПериода = ?(ВходящиеПараметры["from_date"] = Неопределено,
        ТекущаяДата() - 86400,  // по умолчанию за 1 день
        ВходящиеПараметры["from_date"]);

    КонецПериода = ?(ВходящиеПараметры["to_date"] = Неопределено,
        ТекущаяДата(),
        ВходящиеПараметры["to_date"]);

    // Запрос документов
    ЗапросДокументов = Новый Запрос;
    ЗапросДокументов.Текст = "
    |ВЫБРАТЬ
    |    РеализацияУслуг.Ссылка КАК Ссылка,
    |    РеализацияУслуг.Номер КАК Номер,
    |    РеализацияУслуг.Дата КАК Дата,
    |    РеализацияУслуг.Контрагент КАК Контрагент,
    |    РеализацияУслуг.СуммаДокумента КАК Сумма,
    |    РеализацияУслуг.ПолученПредоплата КАК Предоплата,
    |    РеализацияУслуг.Проведен КАК Проведен,
    |    РеализацияУслуг.УстановитьРеквизит("ЗаказСайта") КАК ЗаказСайта,
    |    РеализацияУслуг.УстановитьРеквизит("IDБронирования") КАК IDБронирования,
    |    РеализацияУслуг.УстановитьРеквизит("Беседка") КАК Беседка,
    |    РеализацияУслуг.УстановитьРеквизит("ВремяНачала") КАК ВремяНачала,
    |    РеализацияУслуг.УстановитьРеквизит("ВремяОкончания") КАК ВремяОкончания
    |ИЗ
    |    Документ.РеализацияУслуг КАК РеализацияУслуг
    |ГДЕ
    |    РеализацияУслуг.Дата МЕЖДУ &НачалоПериода И &КонецПериода
    |    И РеализацияУслуг.УстановитьРеквизит("ЗаказСайта") <> ""
    |";

    ЗапросДокументов.УстановитьПараметр("НачалоПериода", НачалоПериода);
    ЗапросДокументов.УстановитьПараметр("КонецПериода", КонецПериода);

    Результат = ЗапросДокументов.Выполнить();
    Выборка = Результат.Выбрать();

    // Формирование JSON
    МассивЗаказов = Новый Массив;

    Пока Выборка.Следующий() Цикл
        Заказ = Новый Структура;
        Заказ.Вставить("order_number", Выборка.ЗаказСайта);
        Заказ.Вставить("document_number", Выборка.Номер);
        Заказ.Вставить("document_date", Выборка.Дата);
        Заказ.Вставить("pavilion_name", Выборка.Беседка);
        Заказ.Вставить("client_name", Выборка.Контрагент.Наименование);
        Заказ.Вставить("start_time", Выборка.ВремяНачала);
        Заказ.Вставить("end_time", Выборка.ВремяОкончания);
        Заказ.Вставить("price", Выборка.Сумма);
        Заказ.Вставить("deposit", Выборка.Предоплата);
        Заказ.Вставить("status", ?(Выборка.Проведен, "posted", "draft"));

        МассивЗаказов.Добавить(Заказ);
    КонецЦикла;

    Ответ.УстановитьКодСостояния(200);
    Ответ.УстановитьТелоИзСтроки(ЗаписатьJSON(Новый Структура("success", Истина, "orders", МассивЗаказов)));

КонецПроцедуры
```

## 2. Настройка экспорта из 1С на сайт

### 2.1. Создание регламентного задания

В 1С:УНФ создайте регламентное задание для периодической выгрузки заказов:

```javascript
Процедура ВыгрузитьЗаказыНаСайт() Экспорт

    // Параметры подключения
    URLСайта = "https://ваш-домен.ru/local/modules/avs_booking/api_1c.php";
    APIКлюч = "секретный_ключ";

    // Получаем заказы за последние 24 часа
    Заказы = ПолучитьЗаказыДляВыгрузки(ТекущаяДата() - 86400, ТекущаяДата());

    Для Каждого Заказ Из Заказы Цикл

        ДанныеЗаказа = Новый Структура;
        ДанныеЗаказа.Вставить("action", "import_status");
        ДанныеЗаказа.Вставить("api_key", APIКлюч);
        ДанныеЗаказа.Вставить("order_number", Заказ.Номер);
        ДанныеЗаказа.Вставить("status", "confirmed"); // или "cancelled"

        HTTPСоединение = Новый HTTPСоединение("ваш-домен.ru", 443);
        HTTPЗапрос = Новый HTTPЗапрос("/local/modules/avs_booking/api_1c.php");
        HTTPЗапрос.Метод = "POST";
        HTTPЗапрос.Заголовки.Вставить("Content-Type", "application/json");
        HTTPЗапрос.Заголовки.Вставить("X-API-Key", APIКлюч);
        HTTPЗапрос.УстановитьТелоИзСтроки(ЗаписатьJSON(ДанныеЗаказа));

        HTTPОтвет = HTTPСоединение.ОтправитьДляОбработки(HTTPЗапрос);

        // Логирование результата
        ЗаписьЖурналаРегистрации("Выгрузка заказа " + Заказ.Номер,
            HTTPОтвет.КодСостояния);

    КонецЦикла;

КонецПроцедуры
```

## 5. Устранение неполадок

| Проблема                               | Решение                                                 |
| -------------------------------------- | ------------------------------------------------------- |
| 401 Unauthorized                       | Проверьте API-ключ в настройках модуля и в 1С           |
| 404 Not Found                          | Проверьте URL HTTP-сервиса в 1С                         |
| Создается, но не отображается на сайте | Нажмите кнопку синхронизации в дашборде                 |
| Нет документов в 1С                    | Проверьте регламентное задание и права доступа          |
| Ошибка сертификата SSL                 | Используйте curl -k для теста или настройте сертификаты |

Регламентное задание запускайте каждые 5-10 минут

Логирование обязательно для отслеживания ошибок

Резервное копирование настроек перед обновлением

Тестовый период перед запуском в production
